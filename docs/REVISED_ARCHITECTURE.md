# Wiki Labs AI Copilot вҖ” Revised Architecture

> **Note**: This document supersedes the original ARCHITECTURE.md where differences exist. Changes are marked with [REVISED] tags. Sections not mentioned here are unchanged from the original architecture.

---

## Overview of Changes

The original architecture is fundamentally sound in its vision, component layering, and technology choices. The following revisions address the critical findings from the architecture review:

| Change | Original | Revised | Reason |
|--------|----------|---------|--------|
| #1 | Skill-per-process MCP servers | Consolidated multi-skill runtime | RAM: 900MB+ вҶ’ 50MB |
| #2 | ChromaDB embedded | SQLite VSS | Stability, zero deps |
| #3 | No key derivation strategy | OS keychain master key | Feasibility |
| #4 | No prompt injection defense | Multi-layer defense | Security |
| #5 | Windows + macOS only | Windows + macOS + Linux | Market reach |
| #6 | Undefined terminal observation | Shell integration (primary) | Feasibility |
| #7 | Simple AiProvider trait | Expanded trait with context mgmt | Completeness |
| #8 | Single-tier observation | Tiered observation pipeline | Latency |
| #9 | Full product scope | Phased MVP (3 phases) | Delivery |

## [REVISED] Data Storage

### Single SQLite Database with VSS

| Store | Technology | Purpose | Change |
|-------|-----------|---------|--------|
| SQLite | Embedded database (single file) | Workspaces, skills, sessions, chat, knowledge, audit logs, memory | **VSS vector extension added** |
| Local Embedding | ONNX Runtime (all-MiniLM-L6-v2) | Generate embeddings locally | **New** вҖ” replaces AI provider embeddings |
| OS Keychain | Windows Credential Manager / macOS Keychain / Linux Secret Service | Encrypted credential storage | Unchanged |
| File System | Local disk | Knowledge source files, exports | Unchanged |

**Why a single database?** Eliminates the dual-write problem between SQLite and ChromaDB. VSS extension adds vector search directly to SQLite. Transactions span both relational and vector data. No synchronization needed.

## [REVISED] Architectural Principles

The original principles remain valid with the following additions:

**8. [NEW] Defense-in-Depth**: All inputs from untrusted sources (observation data, clipboard, screen content) pass through the prompt injection defense layer before reaching the AI provider. AI outputs are validated before being presented to the user.

**9. [NEW] Progressive Disclosure**: The system does not guess. When confidence in intent recognition is low, the system acknowledges uncertainty rather than providing potentially incorrect context. The "Unknown" intent state is a first-class concept.

**10. [NEW] Embedding Independence**: Knowledge embeddings are generated locally, independent of the AI provider. This enables offline semantic search, provider-independent knowledge, and prevents vendor lock-in of the knowledge base.
